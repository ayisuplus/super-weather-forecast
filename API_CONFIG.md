# OpenWeatherMap API 配置指南

## 🌤️ 为什么需要配置API？
当前项目默认使用模拟天气数据，配置真实的OpenWeatherMap API密钥后，将获得：
- ✅ 全球真实天气数据，15分钟更新一次
- ✅ 精准的天气预报数据
- ✅ 中文天气描述
- ✅ 极端天气预警信息
- ✅ 全球地点搜索功能

## 🔑 API密钥申请步骤

### 1. 注册账号
访问OpenWeatherMap官网：https://openweathermap.org/
点击右上角 "Sign In"，注册一个新账号（免费）。

### 2. 获取API密钥
注册登录后：
1. 点击右上角用户名，选择 "My API keys"
2. 在 "Create key" 输入框中填写密钥名称，如 "weather-app"
3. 点击 "Generate" 生成密钥
4. 复制生成的API密钥（类似：`2d131c1e4c8e9d5f7a3b2c1d4e5f6a7b`）

### 3. 免费版限制
- 每分钟最多60次API调用
- 每天最多1000次免费调用
- 提供5天3小时预报
- 对于个人和小流量应用完全足够
- 如果需要更多调用次数，可以升级到付费版

## ⚙️ 项目配置方法

### 方式1：开发环境配置
1. 在项目根目录创建 `.env` 文件（复制 `.env.example` 重命名即可）
2. 填入您的API密钥：
   ```env
   VITE_OPENWEATHER_API_KEY=您的实际API密钥
   ```
3. 重启开发服务器：`npm run dev`
4. 打开控制台，看到 "调用OpenWeatherMap API获取天气数据" 日志说明配置成功

### 方式2：生产环境配置
根据部署平台不同，配置环境变量：
- **Vercel / Netlify**：在平台的环境变量设置中添加 `VITE_OPENWEATHER_API_KEY`
- **Nginx / 服务器部署**：构建前在服务器上创建 `.env` 文件，或在CI/CD流程中注入环境变量
- **Docker部署**：运行容器时通过 `-e VITE_OPENWEATHER_API_KEY=您的密钥` 注入环境变量

### 配置验证
1. 配置完成后运行 `npm run build`
2. 启动应用，点击地球任意位置
3. 打开浏览器开发者工具，查看"网络"标签
4. 看到向 `api.openweathermap.org` 的请求成功，说明配置生效

## 📋 已对接的API列表

| API接口 | 用途 | 调用时机 |
|---------|------|----------|
| `/weather` | 获取指定坐标的当前天气 | 用户点击地球位置时调用 |
| `/forecast` | 获取5天3小时天气预报 | 用户点击位置、切换时间范围时调用 |

## 🎯 智能降级机制
项目已实现完善的降级策略，无需担心API配置错误或调用失败：
1. 如果未配置API密钥，自动使用模拟数据
2. 如果API调用失败（网络错误、密钥错误、调用超限），自动回退到模拟数据
3. 控制台会输出详细日志，便于排查问题
4. 用户体验不受影响，只是数据为模拟生成

## 🚀 开发体验优化
如果您只是开发调试，不需要申请API密钥，项目默认使用模拟数据，可以正常体验所有功能。只有需要真实数据时才需要配置API密钥。

## ❓ 常见问题排查

### Q: API调用返回401错误
A: 检查API密钥是否正确，有没有多余空格或特殊字符。

### Q: API调用返回429错误
A: API调用频率超限，免费版每分钟最多60次，等待1分钟后重试即可，或升级付费版。

### Q: 天气数据不更新
A: OpenWeatherMap数据更新频率为15分钟，相同坐标15分钟内重复调用会返回缓存数据。

### Q: 国内访问速度慢
A: 可以考虑使用代理转发API请求，或使用国内天气数据服务商。
